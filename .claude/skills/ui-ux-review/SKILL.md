# UI/UX review

Start with the user task, not aesthetics.

## Required checks

- Information hierarchy reflects the next user decision.
- Controls have visible labels and clear feedback.
- Loading, empty, error, partial, disabled, and success states are addressed.
- Color is never the sole carrier of meaning.
- Keyboard navigation and visible focus work.
- Text and interactive contrast meet WCAG AA.
- Touch targets are at least 44×44px where touch use is plausible.
- Tables have a contained small-screen strategy.
- Motion communicates hierarchy, feedback, or state and respects reduced motion.
- Existing tokens, spacing, typography, icon family, and component patterns are reused.

Return concrete findings connected to user impact. Do not recommend decorative polish before functional and accessibility issues are resolved.
