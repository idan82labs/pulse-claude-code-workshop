# משימת המשך — ליישם את השיטה בעולם שלכם

בחרו תהליך אמיתי מהעבודה שלכם: פיתוח, QA, DevOps, System, מוצר, נתונים או תפעול. עדיף תהליך שאתם מכירים היטב ושיש בו החלטה אמיתית, לא פרויקט צעצוע.

## התוצר

1. תיאור קצר של התהליך ושל הבעיה שהוא אמור לפתור.
2. יעד ותנאי הצלחה שאפשר לבדוק.
3. Fan-out לארבע נקודות מבט מקצועיות לפחות, Security כשיש trigger, ו-fan-in להחלטה אחת.
4. Skill אישי שאורז ידע מקצועי או תהליך שחוזר אצלכם.
5. תוכנית ארכיטקטורה או תהליך עבודה.
6. חתך עובד אחד.
7. לולאת איכות עם baseline, rubric, target ו-audit לפני, במהלך ואחרי הביצוע — עד שלושה סבבים.
8. README עם ההחלטות, הפשרות, הראיות והצעד הבא שהייתם עושים.

## כלל מרכזי

אל תנסו לבנות מוצר גדול. המטרה היא להראות שאתם יודעים לזהות את הבעיה הנכונה, להגדיר הצלחה, להאציל עבודה, לבקר את התוצאה ולהסביר את ההחלטות שלכם.

## הכנה לראיון

הכינו דמו של שלוש דקות. בסופו, היו מוכנים לענות על השאלות האלה:

- מה Claude ביצע ומה אתם החלטתם?
- איפה המומחים לא הסכימו, ואיך הכרעתם?
- איך מנעתם מהלולאה להתרחב בלי סוף?
- אילו בדיקות מוכיחות שהפתרון עובד?
- איזה סיכון נשאר, ולמה אתם מוכנים לקבל אותו כרגע?
- איזה finding מצא ה-reviewer, איזו ראיה תמכה בו ואיזה verdict התקבל?
