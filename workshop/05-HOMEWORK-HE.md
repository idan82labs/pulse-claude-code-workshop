# משימת המשך — לקחת את המתודה לעולם שלכם

בחרו תהליך אמיתי מהעולם המקצועי שלכם: פיתוח, QA, DevOps, System, מוצר, נתונים או תפעול.

## התוצר

1. תיאור קצר של התהליך והבעיה העסקית.
2. יעד ותנאי הצלחה נצפים.
3. ניתוח מארבע נקודות מבט מקצועיות לפחות.
4. Skill אישי המארז ידע או תהליך חוזר.
5. תוכנית ארכיטקטורה או תהליך עבודה.
6. חתך עובד אחד.
7. לולאת אימות עם עד שלושה סבבים.
8. README הכולל החלטות, פשרות, ראיות ומה הייתם עושים בהמשך.

## כלל מרכזי

אין צורך לבנות מוצר גדול. יש להראות יכולת לזהות את הבעיה הנכונה, להגדיר הצלחה, להאציל עבודה, לבקר את התוצאה ולהסביר את ההחלטות.

## הכנה לראיון

הכינו דמו בן שלוש דקות ותשובות לשאלות:

- מה Claude ביצע ומה אתם החלטתם?
- היכן המומחים לא הסכימו וכיצד הכרעתם?
- כיצד מנעתם מהלולאה להתרחב ללא גבול?
- אילו בדיקות מוכיחות שהפתרון עובד?
- איזה סיכון נשאר ומדוע הוא מקובל כרגע?
